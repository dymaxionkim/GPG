***********************************************   버튼 설명    *****************************************************

1. 토글 버튼 "Input Method"

2. Load Input File

    읽어들일 파라미터 파일을 선택


3. Generate

    기어 형상 생성


4. Save Profile

    생성된 기어 형상 데이타를 *.scr 파일로 저장


5. Save Parameter

    편집한 파라미터들을 추후에 이용할 수 있도록 파라미터 파일로 저장


6. Help

   본 창을 열어줌


7. About ...

    그냥 함 넣어봤음


8. Quit

    종료




********************************************   파라미터 설명    **************************************************

1. Resolution of Involute Curve

기어 측벽 곡선을 몇 개로 분할할 것인지를 결정.
자연수로 입력할 것.
입력 한계 2 ~ 50


2. Resolution of Fillet Curve

이뿌리 필렛 곡선을 몇 개로 분할할 것인지를 결정.
자연수로 입력할 것.
입력 한계 2 ~ 50


3. Module of Gear

기어의 모듈
양수의 부동소숫점으로 입력할 것
입력 한계 0.5 ~ 10


4. Addendum of Gear

기어의 어덴덤 치수
mm단위
양수의 부동소숫점으로 입력할 것
입력 한계 0.1 ~ 10


5. Dedendum of Gear

기어의 어덴덤 치수
mm단위
양수의 부동소숫점으로 입력할 것
입력 한계 0.1 ~ 10


6. Fillet Radius of Gear's Root

기어 이뿌리 필렛의 반경, 또는 기어 창성 공구 첨단부의 둥글기 반경
mm단위
양수의 부동소숫점으로 입력할 것
입력 한계 0.1 ~ 5


7. Pressure Angle

기어의 압력각
Degree 단위
양수의 부동소숫점으로 입력할 것
입력 한계 1 ~ 90


8. Offset Factor

기어의 전위계수.
       0.0의 경우는 전위량이 전혀 없는 '표준 기어'
      조금씩 전위 계수를 높여 가면서 최적 형상을 찾을 수 있음
      기어 측벽의 언더컷 제거 용도로 사용하는 경우가 많음
      너무 수치를 높이면 이빨 두께가 얇아져 기어 이빨의 강도 저하됨
양수의 부동소숫점으로 입력할 것
입력 한계 0 ~ 1


9. Number of Teeth

기어 이빨 갯수
입력 한계 4 ~ 100

********************************************   기어형상 설명    **************************************************

1. 백래쉬 없음
 
따라서 본 도구를 이용하여 생성된 기어는 이빨이 서로 딱딱 물리게 될 것입니다.
백래쉬를 주기 위해서는 Autocad 상에서 Scale 명령어를 이용하여 적절히 조절해 주면 될 것 같습니다.


2. 기어 이뿌리 부분의 필렛은 트로코이드 곡선 방정식을 이용

구면형상의 일반 필렛은 아닙니다.
이를 통하여 기어의 첨단부와 이뿌리 부분이 동작시 간섭되지 않으면서도 
최대한 살두께를 보장하여 기어의 강도를 최적으로 높여줄 수 있습니다.


3. 전위량 설정이 가능

전위계수 변경을 통하여, 기어 이빨 형상을 조정해 줄 수 있습니다.
따라서 기어 측면의 언더컷 등을 제거하는데 도움이 됩니다.


4. 기어 형상 데이타 이용은...  Autocad의 script 명령어를 이용

Autocad 명령줄에서 script 명령어를 치면, 파일 선택 팝업창이 뜨는데...
여기서 본 프로그램에서 저장한 기어 형상 데이타 파일을 찾아서 선택해 주면 스크립트가 실행됩니다.
이때, Osnap 모드는 반드시 해제 상태여야 합니다.  
Osnap 모드가 활성화 된 상태에서는, 괴상한 모양이 그려져 버립니다.
Autocad 버그로 생각됩니다.  
Autocad r14까지는 이런 현상이 없었는데, 2000 이후 부터는 이런 현상이 발생한다고 합니다.
